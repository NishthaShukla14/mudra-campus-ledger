Security Information

Last updated: July 2026

MUD₹A leverages enterprise-grade architecture to ensure your campus funds remain unconditionally secure.

1. Bank-Grade Encryption

All network communications between the web client and the server are secured using HTTPS (TLS 1.3). Passwords are cryptographically salted and hashed using the PBKDF2 algorithm before storage.

2. Biometric Authentication (FaceID)

The MUD₹A Face Unlock mechanism uses a custom AI threshold (Euclidean distance < 0.55) to verify 128 nodal points of your face. This prevents spoofing using 2D photographs and provides a frictionless, high-security login.

3. Transaction Integrity

The backend ledger enforces strict database-level atomic transactions (`db_transaction.atomic()`). This mathematically guarantees that no race conditions can occur during high-concurrency periods (e.g., mass fee payments).

4. Automated Log Audits

Unusual activity, such as rapid repeated failed logins or massive outbound transaction spikes, are automatically flagged by our background cron jobs and forwarded to the Helpdesk for manual administrative review.
