# mudra-campus-ledger
<br>
1.User Authentication: User login/signup karta hai (Manually verified).

2.Dashboard Entry: User ka personalized profile aur balance load hota hai.

3.Transaction Trigger: User "Transfer Money" ya "Fee Payment" action leta hai.

4.Data Validation: Backend check karta hai ki user ke paas sufficient balance hai ya nahi.

5.Ledger Update: Transaction successful hote hi database (SQLite) update hota hai.

6.Analytics Sync: Analytics engine transaction history ko read karke naya graph/chart generate karta hai.

7.Final Confirmation: User ko success/failure ka notification (Toast) dikhta hai.

<br>

Phase	Task Description	Goal
Phase 1: Foundation	Setup Django environment, database schema, aur models.	Stable Backend.
Phase 2: Core UI	Base templates, Navbar, Footer, aur Home page design.	Professional Look.
Phase 3: Auth & Security	Signup/Login logic aur "Admin-Verification" system.	Trusted Access.
Phase 4: Transaction	Transfer money module aur ledger update logic.	Core Functionality.
Phase 5: Analytics	Transaction history ko charts/ledgers mein convert karna.	Data Visibility.
Phase 6: Testing	Edge cases (Zero balance, Invalid user) check karna.	Bug-free System.
Phase 7: Deployment	GitHub push aur Render/Cloud par hosting.	Public Live Access.
